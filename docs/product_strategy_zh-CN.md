# 长桥终端产品战略（2026 上半年）

> 版本：v1.0  
> 适用周期：2026-02-16 ～ 2026-05-22

## 1. 产品愿景

在终端里构建“稳定、顺手、可维护”的投资观察台：  
既能每天看盘，也能作为 Longport OpenAPI + Rust TUI 的工程模板。

## 2. 目标用户

1. 终端重度用户：希望在 SSH / 远程环境轻量看盘。
2. Rust 开发者：希望复用实时行情 + TUI 架构。
3. 运维与值班同学：希望可诊断、可回滚、可审计。

## 3. 核心价值主张

1. **快**：命令行即开即用，不切 GUI。
2. **稳**：限流、重试、单实例、日志容错。
3. **清**：代码结构清晰，文档完整，利于二次开发。

## 4. 北极星指标（North Star）

**WAU-3**：每周至少 3 天活跃的有效用户数。  

原因：比安装量更真实，能衡量“是否形成日常习惯”。

## 5. 关键业务指标

```text
Activation:
  - 首次启动成功率（24 小时内看到行情）

Retention:
  - D1 / D7 留存
  - 预警功能使用率

Reliability:
  - Crash-free sessions
  - API 错误率 / 429 触发率

Efficiency:
  - 启动耗时 P50 / P95
  - 页面切换响应时间
```

## 6. 本期策略

1. 先做激活：`doctor` + 首次引导，降低首次失败率。
2. 再做留存：预警中心 + 工作区记忆，提升回访动机。
3. 最后做增长：反馈闭环 + 发布分轨，形成稳定迭代飞轮。

## 7. 明确不做

1. 自动下单机器人
2. 高频交易能力
3. 机构级托管交易系统

## 8. 风险与守护

1. 需求膨胀：只锁定 P0，P1/P2 进入 backlog。
2. 性能回退：每个里程碑必须做性能基线对比。
3. 发布事故：必须保留回滚版本并演练。
