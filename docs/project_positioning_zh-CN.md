# 项目定位与适用场景（中文）

> 读者对象：第一次接触本项目，想先判断“是否适合我”的开发者、交易研究者、运维同学。

## 1. 我们到底在做什么？

长桥终端是一个基于 Rust 的终端股票应用（TUI），底层通过 Longport OpenAPI 获取行情与账户数据。

它同时承担两个角色：

1. **可直接使用的终端工具**
   - 看自选、看行情、看持仓、看 K 线
2. **可学习的工程样例**
   - 学习 OpenAPI 接入、限流、状态管理、终端渲染

---

## 2. 这个项目解决什么问题？

### 2.1 对使用者

- 不用离开终端，就能快速观察市场与资产状态
- 在 SSH/远程环境下也能工作（只要是交互式 TTY）
- 相比图形端，操作更轻、可脚本化集成更容易

### 2.2 对开发者

- 提供一套“实时行情 + TUI”工程骨架
- 演示了 API 限流、重试、日志容错、单实例治理等实践

---

## 3. 适用与不适用边界

### 适用

- 个人投资观测工作台
- SDK 与接口联调
- 终端实时 UI 开发参考

### 不适用

- 自动交易机器人（策略下单引擎）
- 高频交易场景
- 强合规的机构级托管交易系统

> 设计哲学：先把“稳定可读、可维护、可运维”的基础打牢，再考虑扩展能力。

---

## 4. 核心能力地图

- **行情能力**：自选刷新、搜索、个股详情、K 线
- **资产能力**：账户总览、持仓列表、币种切换
- **运维能力**：单实例锁、日志分目录、发布/回滚脚本
- **稳定性能力**：限流器、退避重试、启动前配置预检

## 6. 版本与命名说明

当前项目命令名已统一为 `changqiao`，并对旧变量保留兼容：

- 新变量：`CHANGQIAO_LOCALE`、`CHANGQIAO_LOG`
- 旧变量：`LONGBRIDGE_LOCALE`、`LONGBRIDGE_LOG`

这保证了老环境不必一次性改完也能平滑迁移。
